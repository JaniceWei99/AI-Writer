# 自媒体平台自动发文能力对比

> 本文档对比主流自媒体平台是否支持通过 API 自动发布文章，为 AI 写作助手的"一键分发"功能提供选型参考。
>
> 最后更新：2026-03-23

---

## 1. 微信公众号

### 1.1 官方 API 能力

微信公众号提供了完整的内容发布 API 链路：

| API              | 说明               | 端点                                      |
|------------------|--------------------|--------------------------------------------|
| 新建草稿         | 创建图文消息草稿   | `POST /cgi-bin/draft/add`                  |
| 修改草稿         | 修改已有草稿       | `POST /cgi-bin/draft/update`               |
| 获取草稿         | 查询草稿内容       | `POST /cgi-bin/draft/get`                  |
| 删除草稿         | 删除指定草稿       | `POST /cgi-bin/draft/delete`               |
| 草稿列表         | 获取草稿总数和列表 | `POST /cgi-bin/draft/batchget`             |
| 发布             | 提交草稿进行发布   | `POST /cgi-bin/freepublish/submit`         |
| 查询发布状态     | 轮询发布结果       | `POST /cgi-bin/freepublish/get`            |
| 删除发布         | 删除已发布文章     | `POST /cgi-bin/freepublish/delete`         |
| 上传图文素材图片 | 上传文章内嵌图片   | `POST /cgi-bin/media/uploadimg`            |
| 上传永久素材     | 上传封面图等       | `POST /cgi-bin/material/add_material`      |

### 1.2 发布流程

```
获取 access_token → 上传素材(封面图) → 新建草稿 → 提交发布 → 轮询发布状态
```

### 1.3 接入要求

- **账号类型**：认证订阅号或认证服务号均可使用草稿箱和发布接口
- **AppID/AppSecret**：需在公众号后台获取
- **access_token**：所有 API 调用需携带，有效期 2 小时，需定期刷新
- **IP 白名单**：需在公众号后台配置服务器 IP

### 1.4 限制与注意事项

- 每天发布次数受账号类型限制（订阅号每天 1 次，服务号每月 4 次群发；发布接口不占群发次数但有频率限制）
- 发布后需经平台审核，审核时间通常几分钟到几小时不等
- 图文消息支持 HTML 富文本，但有标签白名单限制（不支持 `<script>`、`<iframe>` 等）
- 素材图片需先上传到微信服务器，不能直接引用外部链接
- API 调用频率限制：每日 access_token 获取次数有限（2000 次/天），各接口也有独立频率限制

### 1.5 内容格式

```json
{
  "articles": [
    {
      "title": "文章标题",
      "author": "作者",
      "digest": "摘要",
      "content": "<p>HTML 正文内容</p>",
      "content_source_url": "原文链接",
      "thumb_media_id": "封面图素材ID",
      "need_open_comment": 0,
      "only_fans_can_comment": 0
    }
  ]
}
```

### 1.6 适合场景

- 定时自动发布日报/周报
- AI 生成文章后自动推送到草稿箱，人工审核后发布
- 批量文章管理

## 2. 今日头条（头条号）

### 2.1 官方 API 能力

头条号开放平台提供了内容创作 API：

| API        | 说明                       |
|------------|----------------------------|
| 发布文章   | 创建并发布图文文章         |
| 发布图集   | 发布多图内容               |
| 发布视频   | 上传并发布视频             |
| 发布微头条 | 发布短内容（类似微博）     |
| 内容管理   | 查询/修改/删除已发布内容   |
| 数据统计   | 获取阅读量、评论量等       |

### 2.2 发布流程

```
头条号授权(OAuth) → 上传素材(封面/图片) → 调用发布API → 平台审核 → 上线
```

### 2.3 接入要求

- **创作者账号**：需注册头条号并完成资质认证
- **开放平台应用**：在头条号开放平台创建应用，获取 `client_key` 和 `client_secret`
- **OAuth 2.0 授权**：用户需授权应用代为发布
- **内容类目**：需选择内容领域

### 2.4 限制与注意事项

- 所有内容发布后需经过机器审核 + 人工审核
- 审核时间通常 10 分钟到数小时
- 每日发布篇数有上限（通常 5-10 篇，随账号权重调整）
- 内容需原创或有转载授权，平台会进行查重
- API 调用频率有限制

### 2.5 适合场景

- AI 生成文章后自动分发到头条号
- 多平台同步发布
- 数据监控和内容管理

## 3. 小红书

### 3.1 官方 API 能力

小红书开放平台（open.xiaohongshu.com）**目前不提供面向普通创作者的内容发布 API**。

开放平台主要面向：
- **电商商家**：商品管理、订单管理、店铺数据
- **品牌合作**：蒲公英平台（品牌与达人合作）
- **数据服务**：笔记数据分析（仅合作方）

### 3.2 现状

| 能力           | 状态               |
|----------------|---------------------|
| 笔记发布 API   | 不提供             |
| 笔记编辑 API   | 不提供             |
| 笔记数据查询   | 仅合作方/商家      |
| 电商 API       | 面向入驻商家开放   |
| 评论管理       | 不提供             |

### 3.3 第三方方案（有风险）

| 方案                             | 原理                 | 风险                               |
|----------------------------------|----------------------|-------------------------------------|
| 浏览器自动化（Selenium/Playwright） | 模拟用户操作网页版   | 账号封禁风险高，平台有反爬机制     |
| 手机自动化（Appium/ADB）         | 模拟手机 App 操作    | 技术复杂，维护成本高，封号风险     |
| 第三方内容分发工具               | 新榜、蚁小二等       | 部分工具需付费，依赖工具稳定性     |

### 3.4 注意事项

- 小红书对自动化行为检测严格，频繁使用非官方方式发布可能导致**限流或封号**
- 笔记内容有严格的社区规范审核
- 图片是小红书笔记的核心，纯文字笔记曝光率极低
- **建议方案**：AI 生成内容后，通过剪贴板/文件手动复制到小红书 App 发布

## 4. 知乎

### 4.1 官方 API 能力

知乎**不提供面向第三方的内容发布 API**。

知乎开放平台已大幅缩减：
- 曾经有 OAuth 和部分读取 API，但发文 API 从未公开
- 目前开放平台主要面向知乎自身生态内的应用

### 4.2 现状

| 能力           | 状态               |
|----------------|---------------------|
| 发布文章 API   | 不提供             |
| 发布回答 API   | 不提供             |
| 发布想法 API   | 不提供             |
| 读取内容 API   | 极有限（非公开）   |

### 4.3 第三方方案（有风险）

- 存在通过逆向知乎 Web API 实现自动发布的开源项目
- **风险极高**：知乎反爬严格，频繁请求会触发验证码甚至封号
- 知乎内容审核严格，AI 生成内容可能被识别并限流

### 4.4 建议

- 不建议自动化发布到知乎
- 适合 AI 辅助写作 → 手动发布的工作流

## 5. 百家号（百度）

### 5.1 官方 API 能力

百家号开放平台提供了有限的内容管理 API：

| API        | 说明               |
|------------|---------------------|
| 发布文章   | 提交文章内容       |
| 发布动态   | 发布短内容         |
| 内容列表   | 查询已发布内容     |
| 数据统计   | 阅读量等数据       |

### 5.2 接入要求

- 需注册百家号并通过真实性认证
- 需在百家号开放平台申请应用
- API 权限需逐一申请，审核周期较长
- 接入文档相对不完善

### 5.3 限制

- 审核机制严格，尤其对 AI 生成内容
- 每日发布数量有限制
- 百度对内容原创度检测敏感
- 开放平台活跃度较低，更新不频繁

## 6. B站（哔哩哔哩）

### 6.1 官方 API 能力

B站开放平台主要面向视频创作者，图文专栏的 API 支持有限：

| 能力             | 状态                       |
|------------------|----------------------------|
| 视频上传/发布    | 开放平台支持               |
| 专栏文章发布     | 有非公开接口，非官方支持   |
| 动态发布         | 有非公开接口               |
| 评论管理         | 有限支持                   |

### 6.2 专栏文章

- B站专栏（bilibili 专栏）有 Web 端接口，但**未正式开放为公共 API**
- 社区中有通过抓包分析出的接口文档，但属于非官方用法
- 专栏支持富文本和 Markdown（部分）

### 6.3 建议

- 如果目标是发布视频，可使用官方开放平台
- 如果目标是发布图文专栏，建议手动发布
- AI 写作助手可导出为合适的格式，方便用户手动复制

## 7. 抖音

### 7.1 官方 API 能力

抖音开放平台（open.douyin.com）主要面向视频内容：

| API        | 说明                       |
|------------|----------------------------|
| 视频发布   | 上传并发布视频             |
| 图文发布   | 发布图文内容（较新功能）   |
| 内容管理   | 查询/删除已发布内容        |
| 数据开放   | 获取播放量、互动数据       |
| 评论管理   | 评论列表、回复评论         |

### 7.2 接入要求

- 需在抖音开放平台注册应用
- OAuth 2.0 用户授权
- 应用需经过审核
- 部分 API 需要特定权限申请

### 7.3 限制

- 主要面向视频/图文，不适合长文章
- 图文发布为较新功能，API 稳定性待验证
- 内容审核严格

## 8. WordPress

### 8.1 官方 API 能力

WordPress 提供了最完善的内容发布 REST API：

| API        | 说明             | 端点                              |
|------------|------------------|-----------------------------------|
| 创建文章   | 发布/草稿文章    | `POST /wp-json/wp/v2/posts`       |
| 修改文章   | 更新已有文章     | `PUT /wp-json/wp/v2/posts/{id}`   |
| 删除文章   | 删除文章         | `DELETE /wp-json/wp/v2/posts/{id}`|
| 上传媒体   | 上传图片等       | `POST /wp-json/wp/v2/media`       |
| 分类/标签  | 管理分类体系     | `/wp-json/wp/v2/categories`       |

### 8.2 优势

- **完全开放**：REST API 是 WordPress 核心功能，无需额外申请
- **无审核**：自托管站点，内容完全自主控制
- **无频率限制**：取决于服务器性能
- **格式友好**：原生支持 HTML 和 Gutenberg 块编辑器格式
- **认证简单**：Application Passwords、JWT、OAuth 等多种方式
- **生态丰富**：大量插件支持 API 扩展

### 8.3 适合场景

- 个人博客/企业官网自动发布
- AI 生成内容后一键发布
- 内容管理系统 (CMS) 集成

## 接入优先级建议

基于 API 成熟度、稳定性、适合 AI 写作助手的程度，建议接入优先级：

### 第一梯队（推荐接入）

| 平台             | 理由                                         |
|------------------|----------------------------------------------|
| **微信公众号**   | 官方 API 完善，用户基数大，图文生态成熟      |
| **WordPress**    | API 完全开放，零审核，最适合自动化           |
| **今日头条**     | 官方 API 支持，用户量大，内容分发能力强      |

### 第二梯队（可选接入）

| 平台             | 理由                                         |
|------------------|----------------------------------------------|
| **百家号**       | 有官方 API 但文档不完善，百度搜索权重高      |
| **抖音图文**     | API 逐步开放，短内容场景                     |

### 第三梯队（不建议 API 接入）

| 平台             | 理由                                         |
|------------------|----------------------------------------------|
| **小红书**       | 无发文 API，自动化风险高                     |
| **知乎**         | 无发文 API，反爬严格                         |
| **B站专栏**      | 非公开接口，不稳定                           |
| **简书/掘金**    | 无官方 API                                   |

对于第三梯队的平台，建议采用**"AI 生成 + 手动发布"**的工作流：
1. AI 写作助手生成对应平台风格的内容（已支持小红书爆款、公众号文案、头条文案风格）
2. 用户复制内容到对应平台手动发布
3. 或导出为对应格式（Markdown / 富文本）后粘贴发布

## 与本项目的整合建议

AI 写作助手目前已支持多种平台风格的内容生成（小红书爆款、公众号文案、头条文案），后续可考虑：

1. **Phase 1 - 内容适配**（已完成）
   - 各平台风格化内容生成
   - 多格式导出（Word / PDF / Markdown / TXT）

2. **Phase 2 - 一键发布**（规划中）
   - 微信公众号草稿箱对接（优先）
   - WordPress REST API 对接
   - 头条号内容发布对接

3. **Phase 3 - 多平台分发**（远期）
   - 统一发布管理面板
   - 发布状态跟踪
   - 定时发布功能
   - 各平台数据回收（阅读量、互动量）
